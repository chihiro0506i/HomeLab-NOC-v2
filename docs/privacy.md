# プライバシー設計

## 外部通信が発生する場面

完全に外部通信ゼロではない．主に次の場面で外部通信が発生する．

```text
・初回の Docker image 取得
・Pi-hole のブロックリスト更新
・Unbound によるDNS再帰問い合わせ
・Uptime Kuma で外部URLを監視した場合
・NetAlertX や Uptime Kuma で通知連携を設定した場合
```

## 外部に出にくいもの

このテンプレートの初期状態では，自宅LANの端末一覧やDNSログを外部サービスへ積極的に送る設定にはしていない．ただし，通知連携や外部連携を後から設定すると，IPアドレスや端末名などが通知先へ送られる可能性がある．

## Unbound の意味

Pi-hole の上流に Unbound を置くことで，Google DNS や Cloudflare DNS などの外部DNS事業者に問い合わせ履歴をまとめて渡す構成を避ける．ただし，DNSの仕組み上，ルートDNS，TLD DNS，権威DNSなどへの問い合わせは必要に応じて発生する．
