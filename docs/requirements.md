# 要件定義

## 目的

Raspberry Pi 5 を用いて，Pi-hole + Unbound + NetAlertX + Uptime Kuma + Portal を統合した，自宅ネットワーク監視基盤を構築する．初期状態では家全体のDNS設定を変更せず，ラズパイ内サービスの起動確認とローカル監視に限定する．Pi-hole は段階的に自分の端末だけで試し，安全性を確認した後に必要であれば家全体へ適用する．

## 機能要件

```text
Portal
  ・各管理画面への入口を提供する
  ・各サービスの役割を短く説明する
  ・外部APIを使わずローカルHTML/CSS/JSで動く

Pi-hole
  ・DNS問い合わせを受ける
  ・広告・トラッカー系ドメインをブロックする
  ・管理画面からログを確認できる
  ・最初は個別端末だけで試せる

Unbound
  ・Pi-hole の上流DNSとして動作する
  ・ローカル再帰DNSとして名前解決する
  ・特定の外部DNS事業者への依存を減らす

NetAlertX
  ・自宅LAN内の端末を検出する
  ・未知端末や接続変化に気づきやすくする
  ・自宅LAN以外では使わない

Uptime Kuma
  ・Portalや自作Webアプリの死活監視を行う
  ・初期状態では外部サイトを大量監視しない
```

## 非機能要件

```text
安全性
  ・管理画面をインターネットに公開しない
  ・53番ポートを外部公開しない
  ・.env と data/ を公開しない

プライバシー
  ・通知連携は初期状態では使わない
  ・DNSログや端末一覧を外部へ送らない
  ・Pi-hole の上流に Unbound を置く

保守性
  ・起動，停止，状態確認，ログ確認，バックアップをスクリプト化する
  ・設定は .env にまとめる
```
